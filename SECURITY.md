# Security Policy

seizani_app は、写真を端末内で線画化し、星座風画像へ変換する Flutter アプリです。画像処理、ONNX モデル、依存関係、CI 設定に関する脆弱性や公開安全性の問題は、この方針に沿って報告してください。

## Supported Scope

報告対象は、現在の `main` ブランチと公開リポジトリ内の次の領域です。

- Flutter / Dart アプリ本体
- 画像処理フロー、保存・共有対象の生成、端末内推論まわり
- DexiNed / PiDiNet を扱う ONNX 推論コードとモデル取得・生成手順
- 依存関係、GitHub Actions、Issue / PR workflow
- 公開リポジトリに含めるべきではないファイルや情報の混入リスク

ローカル環境だけの設定、未公開のモデル本体、個人の画像データ、公開リポジトリ外のサービスやアカウント設定は、このリポジトリで直接修正できる範囲外です。ただし、このプロジェクトの実装や公開文書が誤って危険な扱いを促している場合は報告対象です。

## Reporting Vulnerabilities And Public-Safety Issues

公開 issue には、問題の種類、影響範囲、再現に必要な最小手順、使用した Flutter / Dart / OS / device 情報を書いてください。悪用可能な詳細、秘密情報、個人画像、未公開データは載せないでください。

公開 issue に詳細を書けない場合は、最小限の概要だけで maintainer の確認を求める issue を作成してください。安全に共有できる範囲が決まるまでは、添付ファイルや詳細な再現データを追加しないでください。

## Do Not Post These In Public Issues

次の情報は public issue、PR、ログ、スクリーンショットに含めないでください。

- 個人写真、顔や人物を識別できる画像、未公開の再現用画像
- EXIF や位置情報を含む画像ファイル
- API key、token、cookie、秘密情報、認証情報
- 個人情報、端末固有 ID、アカウント情報
- ONNX モデル本体、checkpoint、その他の大容量モデルファイル
- 未公開の再現用データ、内部ログ、非公開の検証素材
- 悪用手順をそのまま実行できる詳細な payload や exploit

画像が必要な場合は、権利と公開可否を確認済みの最小サンプル、または合成画像を使ってください。スクリーンショットを載せる場合も、個人情報、ファイルパス、通知、アカウント名、秘密情報が写っていないことを確認してください。

## Areas To Report

画像処理では、個人画像や EXIF が意図せず保存・共有・ログ出力される可能性、端末内処理の境界が誤解される説明、生成物に不要なメタデータが残る可能性を重視します。

ONNX モデルでは、モデル本体の誤コミット、取得元や SHA256 検証の不整合、license / redistribution condition の未確認、モデル生成手順の再現性低下を報告対象にします。このリポジトリではモデルファイルを Git 管理しません。モデルは `assets/models/` にローカル配置し、必要な場合は取得・生成手順と検証値を文書化します。

依存関係では、Flutter / Dart package、Python helper script、GitHub Actions の依存に関する既知脆弱性、古い action、不要な権限、危険なログ出力を報告対象にします。

CI では、秘密情報の出力、モデル本体や build output の混入、テストが公開安全性の前提を壊している状態、PR から実行される workflow の権限が過剰な状態を報告対象にします。

## Severity Guide

- Critical: 秘密情報や認証情報の漏えい、個人画像や EXIF の公開、任意コード実行、公開 workflow からの重大な権限悪用。
- High: モデル本体や未公開データの誤公開、画像処理結果やログに個人情報が残る再現性の高い問題、依存関係の実用的な脆弱性。
- Medium: 公開安全性の誤解を招く文書、SHA256 や取得手順の不整合、CI 権限やログ出力の過剰さ。
- Low: 影響が限定的な文書の曖昧さ、将来の誤運用につながる軽微な設定不備。

緊急度が判断できない場合は、低めに扱わず「影響不明」として報告してください。maintainer 側で影響範囲を確認します。
